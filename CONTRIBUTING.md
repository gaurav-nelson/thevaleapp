# Contributing to TheValeApp

Thank you for your interest in helping improve TheValeApp! While the app itself is not open source, we welcome and appreciate your feedback, bug reports, and feature suggestions.

## How to Contribute

### Reporting Bugs

If you've found a bug or issue with the app:

1. Check if the issue has already been reported by searching [existing issues](../../issues)
2. If not, create a new [Bug Report](../../issues/new?template=bug_report.yml)
3. Fill in all required fields, including your app version
4. Provide clear steps to reproduce the issue
5. Include screenshots or error messages if applicable

### Sharing Feedback

We'd love to hear your thoughts about the app:

1. Create a new [Feedback](../../issues/new?template=feedback.yml) submission
2. Include your app version
3. Be specific about what you like or what could be improved
4. Rate your overall experience if you'd like

### Requesting Features

Have an idea for a new feature or improvement?

1. Check if someone else has already suggested it in [existing issues](../../issues)
2. Create a new [Feature Request](../../issues/new?template=feature_request.yml)
3. Explain the problem your feature would solve
4. Describe how you envision the feature working

## Guidelines

- **Be respectful** - Treat others with respect and kindness
- **Be clear** - Provide detailed information to help us understand your input
- **Be patient** - We review all submissions but may not be able to respond immediately
- **One issue per submission** - Keep each report focused on a single bug, feedback item, or feature

## Finding Your App Version

You can find your app version in the app settings or about section.

## Questions?

If you have questions about using the app, please visit [thevaleapp.com](https://thevaleapp.com/) for documentation and support.

Thank you for helping us make TheValeApp better!
